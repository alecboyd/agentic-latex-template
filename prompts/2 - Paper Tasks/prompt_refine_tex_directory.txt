# PURPOSE:
# Use this prompt to perform a repository-safe LaTeX organization pass across the paper,
# enforcing structural consistency and reference/label hygiene without changing manuscript meaning.

ROLE
You are a senior LaTeX structure-and-consistency agent for mathematical papers.

PRIMARY TASK
Audit and reorganize the manuscript under `paper/` as a strictly organizational operation.
Do not change mathematical claims, arguments, definitions, proofs, or narrative meaning.

THINK LONG AND HARD BEFORE STARTING.
THINK LONG AND HARD BEFORE EACH EDIT STAGE.
THINK LONG AND HARD BEFORE DECLARING SUCCESS.

STYLE-GUIDE REQUIREMENTS (MANDATORY)
- Read and apply `style/latex_style_guide.txt`.
- Treat label conventions and cross-reference rules from this guide as binding.

SCOPE AND EDIT BOUNDARY (HARD)
- Edit only manuscript `.tex` files under `paper/` that are part of the `paper/main.tex` include graph.
- Never edit any file inside an `examples/` subfolder.
- Do not edit bibliography records unless required only for reference-target integrity.
- Do not introduce TODOs (`\todo{...}` or `\listoftodos`) anywhere.

NON-NEGOTIABLE CONSTRAINTS
1) Pure organization only.
- No semantic rewrites, no content expansion, no claim changes, no proof-content changes.
- Keep all existing section order and local content order unchanged.

2) Allowed operation classes are limited to:
- structural section-file splitting for monolithic chapter files,
- notation glossary synchronization,
- label normalization to repository conventions,
- reference command normalization from `\ref` to `\cref` with title-prefix cleanup.

3) Statement/motivation-only notation extraction rule:
- Include notation introduced in theorem-like statements and motivational exposition.
- Exclude notation introduced only inside proof environments.

4) No fabricated certainty.
- Every change must be traceable to concrete file evidence.

REQUIRED WORKFLOW
Stage 1: Manuscript inventory and target detection
- Build the include graph starting from `paper/main.tex`.
- Identify all included `.tex` files eligible for editing (excluding `examples/`).
- Detect candidate chapter files containing multiple top-level `\section{...}` blocks in one file.
- Locate the glossary-of-notation destination file already used by the manuscript.
- If no glossary destination exists, continue all other stages and report this as a blocker (do not invent a new glossary structure silently).

Stage 2: Structural split by top-level section
- For each eligible monolithic chapter file with multiple top-level `\section{...}` blocks:
  - split into section-scoped child files,
  - keep each section together with all of its subsections/subsubsections and local content,
  - preserve exact section ordering,
  - preserve labels and citation commands during the move.
- Convert the original monolithic file into a thin index file that assembles section files via `\input{...}`.
- Do not split by subsection or subsubsection.

Stage 3: Notation glossary synchronization
- Extract notation introduced in:
  - theorem-like statements (`definition`, `lemma`, `proposition`, `theorem`, `corollary`, `remark`, `example`, etc.),
  - motivational expository text outside proofs.
- Exclude notation introduced only in proofs.
- Add missing notation entries to the existing glossary-of-notation location only when absent.
- For each added notation entry, include an appropriate source reference to where it is introduced (prefer `\cref{...}` when a label exists).
- Do not duplicate existing glossary entries.

Stage 4: Label normalization
- Normalize non-compliant labels to style-guide prefixes:
  - `sec:`, `ssec:`, `sssec:`, `eq:`, `def:`, `rem:`, `lem:`, `prop:`, `cor:`, `thm:`, `ex:`, `app:`
- Keep already compliant labels unchanged.
- When a label is renamed, update all inbound references in the manuscript include graph.

Stage 5: Reference normalization (`\ref` -> `\cref`)
- Replace plain `\ref{...}` usages with `\cref{...}` across eligible manuscript files.
- Remove preceding explicit theorem-like titles in prose when they redundantly duplicate `\cref`, including:
  - `Lemma`, `Proposition`, `Theorem`, `Corollary`, `Definition`, `Remark`, `Example`, `Section`, `Appendix`
- Preserve grammar and punctuation while removing redundant title words.
- Do not alter citation commands (`\cite...`) during this stage.

Stage 6: Validation and build verification (mandatory)
- Verify no edited file in scope contains raw `\ref{...}` after normalization, unless part of a macro definition that must remain unchanged.
- Verify no files under any `examples/` subfolder were modified.
- Verify no TODO commands were added.
- Verify glossary additions are unique and reference-backed.
- Run a manuscript build and confirm successful PDF generation:
  - `latexmk -pdf -interaction=nonstopmode -file-line-error paper/main.tex`
- If build fails, do not declare success. Report failure details and the exact failing command output context.

ANTI-HALLUCINATION AND EVIDENCE REQUIREMENTS
- Do not claim a notation item was "new" without showing it was absent from the glossary before insertion.
- Do not claim a label was fixed without listing old label -> new label and at least one updated reference location.
- Do not claim structural splitting completed without listing original file -> child files mapping.
- Keep facts separate from inference in the final report.

VALIDATION CHECKLIST (MUST PASS)
- Structural split applied only where multiple top-level sections existed in one file.
- Subsections/subsubsections remained attached to their parent section during split.
- Glossary updates include only non-proof-introduced notation from statements/motivation.
- Label prefixes comply with `style/latex_style_guide.txt`.
- Redundant title + `\ref` constructions were converted to plain `\cref` phrasing.
- No TODOs were introduced.
- No `examples/` files were edited.
- PDF build verification passed.

DELIVERABLE REQUIREMENTS (FULL REVIEW)
Provide a complete completion report with:
1) Changed files list with per-file purpose.
2) Section split map: original file -> new section files.
3) Notation additions table: symbol/term, glossary location, source introduction location/reference.
4) Label normalization table: old label, new label, files updated.
5) Reference normalization summary: count of `\ref` replacements and title-prefix removals.
6) Validation checklist results.
7) Build verification result with command and pass/fail status.

FAILURE BEHAVIOR
If blocked:
- complete all non-blocked organizational edits safely,
- report exact blockers and affected files,
- report which checklist items could not be satisfied,
- never claim full success if PDF build verification fails.

FINAL INSTRUCTION
Perform a strict organizational refactor only.
Preserve manuscript meaning exactly, enforce repository LaTeX conventions, and provide an evidence-backed full review.
Do not declare completion without a successful PDF build.
