LATEX STYLE GUIDE
Repository: agentic-latex-template
Scope: all AI edits to LaTeX content and LaTeX support files

==================================================================
1) GOAL OF THIS GUIDE
==================================================================
This guide is about making high-quality LaTeX contributions, not just changing files.
When editing, optimize for:
- typographic quality
- mathematical clarity
- stable cross-references
- low-friction maintainability
- minimal surprise for future editors

If this guide conflicts with repository code, code is the source of truth.

==================================================================
2) FOLDER ROLES (USE THESE DIRECTLY)
==================================================================
- paper/chapters: narrative document content
- paper/figures: figure wrappers (layout/caption context)
- paper/elements: reusable TikZ drawing components
- paper/style.sty: theorem/reference/style conventions
- paper/extra.sty: helper macros, TikZ setup, TODO system

Dependency direction:
- chapters -> figures
- figures -> elements
- elements -> no chapter or figure imports

Hard safety rule:
- NEVER modify anything inside any examples subfolder unless explicitly requested.
  Examples:
  - paper/chapters/examples/
  - paper/figures/examples/
  - paper/elements/examples/

==================================================================
3) MATH WRITING QUALITY RULES
==================================================================
- Keep display math semantically tight; avoid prose inside equations unless necessary.
- Prefer one idea per displayed equation.
- Use aligned environments for multi-line derivations instead of manual spacing hacks.
- Do not use inline math for long expressions that hurt line breaks.
- Keep notation consistent with existing symbols in the current document.
- Prefer semantic commands/macros already defined in the repo over raw symbol repetition.

Punctuation convention around equations:
- Avoid sentence punctuation inside \[...\] and equation environments unless explicitly requested.

==================================================================
4) MACRO POLICY
==================================================================
Create macros when they reduce repeated medium-to-large editing burden.
Good candidates:
- repeated structured expressions
- repeated formatting idioms
- repeated diagram style fragments

Do NOT create macros for one-off symbols or trivial single-use shorthands.
When adding macros:
- place them in the correct style/helper file (usually paper/extra.sty)
- choose descriptive names
- avoid collisions with standard LaTeX package commands
- prefer backwards-compatible additions over breaking renames
- add comments / documentation to the macros

==================================================================
5) CROSS-REFERENCING RULES
==================================================================
Use stable labels and reference through \cref by default.

Label prefixes:
- sec:, ssec:, sssec:
- eq:
- def:, rem:, lem:, prop:, cor:, thm:, ex:
- app:

Reference quality rules:
- prefer \cref{...} over raw \ref{...} for theorem-like and equation references
- do not rename existing labels unless required for correctness
- when moving content, preserve label IDs whenever possible
- write labels with dashes between words, example "sec:measure-theory"

==================================================================
6) DRAFT MARKER USAGE RULES
==================================================================
Supported forms:
- \todo{...}
- \listoftodos
- \prelim{description}{reference}
- \listofprelims
- \hidetodos / \showtodos
- \hideprelims / \showprelims

Authoring rules:
- keep TODO text actionable and concise
- use TODOs for editing tasks, not long explanations
- use \listoftodos when a consolidated task list is needed
- use preliminary markers for external background facts, source checks, or bibliographic references that should remain visible during drafting
- use \listofprelims when a consolidated preliminary marker list is needed
- use hide toggles in the preamble when draft markers should not appear in the rendered PDF

Inline spacing rule (important):
- When inserting \todo or \prelim in a sentence, place the \ directly next to the previous word without a space to avoid extra spacing in the output

==================================================================
7) FIGURE CONTRIBUTION RULES
==================================================================
- Put layout/context composition in paper/figures.
- Put reusable drawing primitives in paper/elements.
- Do not duplicate large TikZ blocks across multiple files; extract shared parts.
- Keep line widths, node sizing, and label density visually consistent within a figure set.
- Prefer descriptive node/style names over opaque short names when adding new complex diagrams.

==================================================================
8) BIBLIOGRAPHY AND CITATION RULES
==================================================================
- Keep bibliography entries in paper/refs.bib.
- Use stable citation keys.
- Prefer existing citation style behavior; do not change backend/style configuration unless requested.
- When editing prose, ensure every new citation key resolves.
- Add comments and documentation for each citation you add.

==================================================================
9) COMPILATION AND VALIDATION
==================================================================
Before finalizing LaTeX edits:
1. Compile and confirm no new hard errors.
2. Check for broken references from moved/renamed labels.
3. Check that new TODO insertions did not degrade line spacing.
4. Confirm includes/inputs follow chapters -> figures -> elements layering.
5. Avoid unrelated formatting churn in untouched sections.

==================================================================
10) AI CONTRIBUTION STANDARD
==================================================================
Every LaTeX change should be:
- correct
- consistent with existing conventions
- minimal but complete
- easy for a human collaborator to continue from

Do not optimize for cleverness; optimize for document quality and maintainability.
